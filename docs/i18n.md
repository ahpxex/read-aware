# Internationalization (i18n)

ReadAware ships a full i18n system built on **i18next + react-i18next**. Eight
locales are supported: `en`, `zh-Hans`, `zh-Hant`, `ja`, `fr`, `de`, `ru`, `es`.
English is the source of truth and the fallback; the other seven are
machine-translated first passes pending human review.

## Why i18next

Chosen over Lingui / react-intl / a custom layer because it lands every
requirement on a first-class path for this stack: typed keys via module
augmentation, native CLDR plurals (Russian one/few/many/other via
`Intl.PluralRules` — no ICU add-on, so the runtime stays ~18 KB), per-locale lazy
loading through Vite's dynamic-import splitting, and a clean shared-UI story
(`@read-aware/ui` reads the singleton via a `react-i18next` peer dep, importing
zero app state).

## Layout

```
apps/web/src/i18n/
  config.ts        LOCALES, NAMESPACES, AppLocale type, LOCALE_LABELS, resolveLocale()
  instance.ts      the single i18next instance (no init — avoids an import cycle)
  index.ts         initI18n() + setLocale() + useLocale(); re-exports useTranslation/Trans/config/format
  detect.ts        first-run locale detection (stored pref → navigator → en)
  format.ts        locale-aware Intl layer (dates, numbers, %, durations, weekday/month names, hours)
  i18next.d.ts     typed-key augmentation, derived from the en catalogs
  locales/<lng>/<ns>.json   catalogs — 8 locales × 9 namespaces
```

Namespaces map to product surfaces: `common`, `ui`, `settings`, `reader`,
`shelf`, `stats`, `ai`, `command`, `nav`. Each is its own lazy chunk, so a route
only loads the namespaces it renders, and only the active locale ships.

## Boot & switching

`main.tsx` reads the persisted language (or auto-detects on first run) and awaits
`initI18n(locale)` **before** the app mounts, so the first paint is translated —
no flash, no missing-key render. The picker lives in Settings → General; changing
it calls `setLocale()` (runtime `changeLanguage` + `<html lang>` sync) and
persists to `generalSettings.language`. A `null` language means "not yet chosen"
→ auto-detect.

## Using it

```tsx
import { useTranslation } from "../../../i18n"; // app code: the i18n barrel

function Thing() {
  const { t } = useTranslation("reader");        // namespace-scoped
  return <button aria-label={t("nextPage")}>{t("next")}</button>;
}
```

- **Interpolation:** `t("greeting", { name })` ← `"Hello {{name}}"`.
- **Plurals:** `t("books", { count })` ← author `books_one` / `books_other` in en;
  translators regenerate the categories per language.
- **Rich text:** `<Trans i18nKey="…">` with a `components` map (see AIConfigPanel).
- **Non-component modules** (command builders, label maps) take a
  `t: TFunction<"ns">` param from their caller — never a global.
- **`@read-aware/ui`** imports `useTranslation` from `react-i18next` directly (a
  peer dep) and must never import from `apps/web`.

## Formatting (not just strings)

Anything locale-sensitive goes through `i18n/format.ts` so it follows the *app*
language, not the OS. `formatDate`, `formatNumber`, `formatPercent`,
`formatDuration`, `formatHour`, `getWeekdayNames`, `getMonthNames`,
`formatRelativeDays`. These read the active locale internally, so call sites stay
parameterless; a component that renders formatted values with no `t()` nearby
should still call `useTranslation()` so it re-renders on a language switch.

## Adding a string

1. Add the key + English value to `locales/en/<ns>.json`.
2. Use `t("your.key")` (typed — the build fails on an unknown key).
3. Add translations to the other seven `locales/<lng>/<ns>.json`, or leave blank
   to fall back to English.
4. `bun run i18n:extract` (i18next-parser) can scaffold/merge keys from source.

## Adding a locale

Add the BCP-47 tag to `LOCALES` + a `LOCALE_LABELS` autonym in `config.ts`, create
`locales/<tag>/*.json`, and (if its script needs it) extend the font stack.

## Known follow-ups

- **Machine translations need a human pass** — especially Russian grammar and
  terse strings (e.g. `stats.achievements.next`). Structure/plurals/placeholders
  are validated; wording is not.
- **CJK fonts:** the stack is Inter/serif; bundle Noto Sans SC/TC/JP fallbacks so
  `zh-*`/`ja` render well.
- **Untranslated engine edges:** `reader/lib/foliate-engine.ts` load-failure
  strings and the `useReaderSession` missing-file throw pass through raw (the
  vendored-engine loader boundary makes threading `t` invasive).
- **`ui` Progress** keeps a plain `NN%` (numeric percent not run through Intl).
- **Storybook** initializes i18n in `en` via a preview loader; there is no
  per-story locale switcher yet.
