二、缺口清单

第一档:明显的能力空洞(建议尽快补)

  • 没有事件系统。插件完全是被动的:感知不到"翻开了一本书 / 划了一条高亮 / 读完一章 / 选中了文字"。做不了自动化类插件(自动同步、阅读提醒、读完统计)。这是和 Obsidian 差距最大的一块——建议 ctx.events.on("book-opened" | "annotation-created" | "selection-changed" | …),返回 disposable。
  • 读不到书的内容。agent 有 bookText 端口(目录 + 章节文本),插件没有。"导出全书笔记带上下文"、"全文分析"类插件做不了。建议 ctx.reading.getToc / getChapterText(挂 reading-data)。
  • 没有阅读器导航 API。插件拿得到 CFI 却不能跳转过去。生词本的"回到出处"、书签类插件都被卡住。建议 ctx.reader.goTo(cfi | href)。
  • 应用退出时不调 deactivate。只有禁用/卸载才调;插件没有可靠的收尾时机。

第二档:体验/完整性短板

  • 表单太薄:无多行文本、数字、校验错误回显(onSubmit 无法把错误标到字段上)。
  • 标注写入不完整:能建/删,不能改(updateNote、recolor);书元数据只读(合理,但笔记编辑是真实需求)。
  • 插件命令进不了快捷键系统(ShortcutId 是闭合 union)。用户没法给"收藏生词"绑键。
  • 无声明式"插件设置页":插件只能自己用命令 + 表单模拟(Word Bank 就这么干的),设置项散落。建议 manifest 里声明 settings 表单,聚合进设置 → Plugins。
  • llm.ask 无流式、无档位选择;工具无法按书/线程粒度启停(全局开关)。
  • zip 安装未做(只有文件夹);插件更新无主动提醒(要自己开市场页看)。
  • 打包构建未验证:全部实测在 dev CSP 下,生产 CSP 的 raplugin: 还欠一次 bun run build:desktop 实测。

第三档:记录在案,不急

  • push 式上下文注入(设计文档里已明确"等真实需求");插件 i18n;开发热重载;市场的截图/分类/统计;jsDelivr 约 12h 缓存延迟;溢出菜单里隐藏挂件触发器的 CSS 方案偏 hack(依赖 Popover 内部结构);图标双源问题的根治(真实按钮也从注册表取图标)。

三、一句话结论

骨架是完整且对的:挂载点、权限、市场、TS 工具链、用户摆放权都闭环了。真正的下一步不是再加界面能力,而是把插件从"被动应答者"变成"有感知的参与者"——事件系统 + 书内容读取 + 阅读器导航,这三样补上,自动化类和深度阅读类插件才写得出来。
