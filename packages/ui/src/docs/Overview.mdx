# ReadAware Design System

ReadAware is an editorial reading application. The design language draws from print
publishing: warm paper tones, generous whitespace, serif display type, and extreme
restraint.

## Principles

**Restraint over decoration.** Every element earns its place. If it does not serve
comprehension, it does not belong.

**Print-inspired.** The interface borrows from book and magazine design -- quiet
surfaces, typographic hierarchy, and open margins.

**Warm and quiet.** A paper-toned palette replaces the cold whites and blues of
typical software. The result feels approachable without being loud.

**Typography does the work.** Size, weight, and spacing carry hierarchy. The
interface stays visually spare because the type system handles structure.

## Restraint in practice

- **Badge and Tag exist, but stay quiet.** They are hairline-bordered, stone-toned
  metadata markers -- never colored pills or attention magnets.
- **Color is functional, never ornamental.** The two accents in the system are
  state colors: emerald for success (`Alert` / `Toast` success variants) and red
  for the destructive `Button` `danger` variant. Errors themselves stay in the
  stone palette -- the `destructive` variants of `Alert` and `Toast` use a firmer
  border and full-value title, not a red tint.
- **One shadow in the whole system.** `Toast` floats over content and carries a
  `shadow-sm`; everything else separates with hairline borders and fills.
- No gradients, no ornamental highlights, no decorative icons.

## When in doubt

Choose the quieter option. If two approaches achieve the same goal, pick the one
with less visual weight.

## Iconography

- Use `@phosphor-icons/react` for all UI icons.
- Avoid hand-authored inline SVG icons in feature code.
- Icons should be functional, not decorative.

## Story organization

- **`Design System/…`** -- stories and docs from this package (`@read-aware/ui`):
  components, typography, and these guideline pages.
- **`Interface/…`** -- feature stories co-located in `apps/web` (shelf, reader,
  context panel, …) that compose the design system into product surfaces.
