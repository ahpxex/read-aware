# Typography

ReadAware uses two typefaces: **Inter** (sans-serif) for UI and body text, and the
system **serif** stack for display headings.

## Type scale

| Component | Font | Size | Weight | Tracking | Leading | Color |
|-----------|------|------|--------|----------|---------|-------|
| Display | Serif | 5xl / 6xl / 7xl | Normal | Tight | 0.98 | `text-fg` |
| Heading | Sans | xl / 2xl / 3xl / 4xl | Semibold | Tight | Default | `text-fg` |
| Body | Sans | base / lg | Normal | Default | 2rem | `text-fg` |
| Eyebrow | Sans | 11px | Medium | Default | Default | `text-fg-muted` |
| Caption | Sans | 12px | Normal | Default | Default | `text-fg-muted` |

Colors are the semantic tokens from the Colors guideline, so the whole type
system adapts to the dark theme for free -- no `dark:` variants needed.

## Design tokens

Font sizes live in Tailwind v4's `--text-*` theme namespace (there is no
`--font-size-*` namespace):

```
--text-eyebrow: 0.6875rem   (11px)
--text-caption: 0.75rem     (12px)
--leading-display: 0.98
--leading-body: 2rem
```

## Guidelines

- **Display** is reserved for hero-level headings. One per view maximum.
- **Eyebrow** labels are small, quiet section and metadata labels (sentence case, no wide tracking).
- **Body** text uses generous line height (2rem) for comfortable reading.
- Avoid mixing serif and sans within the same text block.
- Never use bold for emphasis in body text -- use structure (headings, eyebrows)
  to create hierarchy instead.
